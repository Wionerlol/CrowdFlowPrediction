# 静态特征维度规格

**版本**: v1.4  
**日期**: 2026-06-17（v1.0 为 2026-06-13）  
**目标**: 为每个网格节点定义固定的静态属性向量，作为 GNN 节点初始嵌入

**变更说明（v1.0 → v1.1 → v1.2 → v1.3 → v1.4）**：
- v1.1：新增 2.2 距离特征（2维）；路网 4维→5维；总维度 33→36
- v1.2：transport 类从 POI 中剥离，POI 12维→11维；新增 2.7 公共交通特征（2维，地铁站密度 + 地铁入口密度）；总维度 36→37维
- v1.3：公共交通特征追加公交站密度（highway=bus_stop），2维→3维；总维度 37→38维
- v1.4：新增 2.8 轨道线路特征（num_subway_lines + is_transfer_hub，2维）和 2.9 公交线路特征（num_bus_routes，1维）；总维度 38→**41维**

---

## 一、静态特征的作用

动态特征（人流时序）表达"发生了什么"，静态特征表达"这个地方是什么"。
在 STAEformer/TESTAM 等自适应嵌入模型中，静态特征可替代或增强可学习节点 ID 嵌入，使模型在城市迁移时保持可解释性。

## 二、TaxiBJ 32×32 网格节点特征

每个节点 (row, col) 对应一个约 1.33km×1.73km 的格子，静态特征向量维度合计 **41维**：

### 2.1 地理位置特征（4维）

| 索引 | 特征 | 取值 | 计算方式 |
|------|------|------|---------|
| [0] | 归一化纬度 | [0,1] | `(lat_center - 39.75) / 0.50` |
| [1] | 归一化经度 | [0,1] | `(lon_center - 116.25) / 0.50` |
| [2] | 到城市中心距离（归一化） | [0,1] | Haversine 到天安门(116.3974, 39.9087)，min-max |
| [3] | 环路区域 ID（归一化） | [0.2,1.0] | 1~5（二环内~五环外）按距离分桶，除以 5 |

```python
RING_THR_KM = [3.5, 7.0, 11.0, 16.0]   # 二/三/四/五环边界
ring_id = 1  # 二环内
for i, thr in enumerate(RING_THR_KM):
    if dist_center_km > thr:
        ring_id = i + 2
```

### 2.2 距离特征（2维）

| 索引 | 特征 | 取值 | 计算方式 |
|------|------|------|---------|
| [4] | 到最近交通枢纽距离（归一化） | [0,1] | Haversine 到 4 个枢纽最小值，min-max |
| [5] | 到最近商业中心距离（归一化） | [0,1] | Haversine 到 4 个商业中心最小值，min-max |

**北京枢纽**：北京站、北京西站、北京南站、首都机场 T3  
**北京商业中心**：王府井、三里屯、中关村、西单

### 2.3 路网特征（5维，来自 OSM）

从 `beijing-latest.osm.pbf` 提取，按网格 bbox 空间裁剪，投影至 UTM EPSG:32650 计算长度，log1p 变换后 z-score 归一化：

| 索引 | 特征 | 描述 |
|------|------|------|
| [6] | 路网总密度 | 格内所有道路总长 / 格面积（km/km²） |
| [7] | 主干道密度 | motorway/trunk/primary/secondary 长 / 格面积 |
| [8] | 地方道密度 | tertiary/unclassified/residential 长 / 格面积 |
| [9] | 服务道密度 | service 类道路长 / 格面积 |
| [10] | 慢行道密度 | footway/path/cycleway/pedestrian/steps 长 / 格面积 |

**实测统计（北京）**：路网总密度均值 10.0 km/km²，共提取 90,380 条道路。

> **说明**：原规格中"交叉口密度"和"是否含地铁站"两维因数据未就绪而暂未实现。地铁站二值特征待公共交通数据调研后补充（见待办事项）。

### 2.4 POI 类别密度（11维，来自 OSM）

从 OSM 节点提取，按 **11 类功能性 POI** 分类后聚合到网格，取密度（个/km²）做 log1p 变换后 z-score 归一化。

> **transport 类已剥离**：公交站/地铁入口等交通基础设施归入 2.7 公共交通特征，与功能 POI 分开建模。这与 PDF 原始需求（餐饮/购物/娱乐/办公/住宅）精确对齐，并消除了北京 transport 类主导（原占65%）导致 POI 相似度退化的问题。

| 索引 | POI 类别 | 主要 OSM Tag |
|------|---------|-------------|
| [11] | 餐饮 | `amenity=restaurant/cafe/fast_food/bar` |
| [12] | 购物 | `shop=supermarket/mall/clothes/...` |
| [13] | 娱乐 | `leisure=cinema`, `amenity=theatre/nightclub` |
| [14] | 办公商业 | `office=*`, `amenity=bank/coworking_space` |
| [15] | 居住 | `building=apartments/residential` |
| [16] | 教育 | `amenity=school/university/college` |
| [17] | 医疗 | `amenity=hospital/clinic/pharmacy` |
| [18] | 政务机构 | `amenity=townhall/police/courthouse` |
| [19] | 旅游景点 | `tourism=*` |
| [20] | 体育 | `leisure=sports_centre/stadium/gym` |
| [21] | 宗教 | `amenity=place_of_worship` |

**实测统计（北京）**：共提取 12,536 个功能性 POI，每格平均 12.2 个。

### 2.5 卫星遥感特征（5维，来自 Landsat-8）


L1TP 级 DN 值 → TOA 反射率（MTL.txt 校正参数），裁剪至研究区后重投影至 WGS84（25m 分辨率），按网格聚合像素均值，z-score 归一化：

| 索引 | 特征 | 计算公式 | 含义 |
|------|------|---------|------|
| [22] | NDVI | `(B5-B4)/(B5+B4)` | 植被指数，[-1,1] |
| [23] | NDBI | `(B6-B5)/(B6+B5)` | 建筑密度指数，正值=城区 |
| [24] | MNDWI | `(B3-B6)/(B3+B6)` | 水体指数，正值=水域 |
| [25] | B4 均值 | 红波段平均 TOA 反射率 | 建筑/裸地亮度 |
| [26] | B4 标准差 | 格内像素标准差 | 土地利用异质性 |

**实测统计（北京，2015-04-16）**：NDVI 均值 0.219，NDBI 均值 -0.022，太阳高度角 54.80°。

### 2.6 时间不变历史统计特征（8维）


基于训练集（BJ13+BJ14+BJ15前75%）的 inflow 时序计算：

| 索引 | 特征 | 计算方式 | 归一化 |
|------|------|---------|-------|
| [27] | 历史均值 | 全时段 inflow 均值 | min-max |
| [28] | 历史峰值比 | 该格 p99 / 全局 p99 | 直接使用 [0,1] |
| [29] | 历史零值率 | inflow=0 的时步比例 | 直接使用 [0,1] |
| [30] | 工作日峰均比 | 工作日 7~8 点均值 / 工作日全天均值 | clip [0,5] |
| [31] | 周末/工作日比 | 周末均值 / 工作日均值 | clip [0,5] |
| [32] | 夜间占比 | 0~5 点均值 / 全天均值 | clip [0,1] |
| [33] | 净流向 | mean(inflow - outflow) | min-max |
| [34] | 流量变异系数 | std(inflow) / mean(inflow) | clip [0,5] |

### 2.7 公共交通特征（3维，来自 OSM）

从 OSM PBF 提取地铁站点、地铁入口和公交站，聚合到网格，log1p 变换后 z-score 归一化：

| 索引 | 特征 | OSM Tag | 北京 | 纽约 |
|------|------|---------|------|------|
| [35] | 地铁站密度（个/km²） | `station=subway` | 369 站，253/1024（24.7%）格覆盖 | 238 站，41/75（54.7%）格覆盖 |
| [36] | 地铁入口密度（个/km²） | `railway=subway_entrance` | 983 个入口，208/1024 格覆盖 | 1,153 个入口，41/75 格覆盖 |
| [37] | 公交站密度（个/km²） | `highway=bus_stop` | 10,831 站，787/1024（76.9%）格覆盖，均值 4.56/km² | 3,015 站，66/75（88.0%）格覆盖，均值 12.57/km² |

> 三维分工：地铁站数反映地铁线网覆盖，地铁入口数反映换乘通达性，公交站数补充地面公交覆盖（地铁盲区的唯一公共交通选项）。三者合计完整刻画区域公共交通可达性。

### 2.8 轨道线路特征（2维，来自政府数据 + OSM）

通过站名匹配（政府轨道站点信息.xlsx ↔ OSM subway 节点）生成每格穿越的地铁线路集合：

| 索引 | 特征 | 取值 | 计算方式 |
|------|------|------|---------|
| [38] | 穿越地铁线路数（log1p） | [0,~] | 格内唯一线路数，log1p 变换 |
| [39] | 是否换乘枢纽 | {0, 1} | 格内存在 ≥2 条线路交叉站点 |

> 与 [35] 地铁站密度的区别：密度反映"有多少个站"，线路数反映"通达多少条线路"。双向信息互补，对换乘枢纽（如西直门、国贸）的流量峰值有更强解释力。

**实测统计：**

| | 北京 | 纽约 |
|--|------|------|
| 有地铁线路的格数 | 208/1024（20.3%） | 41/75（54.7%） |
| 换乘枢纽格数（≥2条线） | 55 | 33 |
| 数据来源 | 政府轨道站点信息.xlsx + OSM 坐标，匹配率 940/1079（87.1%） | OSM route=subway 关系，31条线路 |

### 2.9 公交线路特征（1维，来自 MTA GTFS / OSM route=bus 关系）

通过解析公交线路数据，直接获取每个公交站所属线路：

| 索引 | 特征 | 取值 | 计算方式 |
|------|------|------|---------|
| [40] | 网格内公交线路数（log1p） | [0,~] | 格内所有公交站覆盖的唯一线路数之并集，log1p 变换 |

**实测统计：**

| | 北京 | 纽约 |
|--|------|------|
| 有公交线路的格数 | 765/1024（74.7%） | 48/75（64.0%） |
| 网格均值 | 7.6 条 | 7.6 条 |
| 网格最大值 | 64 条 | 41 条 |
| 数据来源 | OSM 2098条关系（覆盖率88.4%）+ 政府公交站点信息.xlsx补全（+890站） | MTA GTFS 5区数据（255条bus线路），**100% 覆盖** |

## 七、特征重要性预判

按对人流预测的预期贡献排序（基于文献）：

1. **历史统计特征**（8维）— 直接反映流量模式，影响最大
2. **POI 密度**（11维）— 决定区域功能，对早高峰/晚高峰幅度影响大
3. **路网特征**（5维）— 决定区域通达性，对拥堵传播建模重要
4. **距离特征**（2维）— 捕捉城区中心性效应（离枢纽越近人流越大）
5. **地理位置**（4维）— 捕捉城区方位效应（中心 vs 边缘）
6. **公共交通特征**（6维，密度3+线路3）— 地铁+公交可达性及换乘通达度，对地铁盲区流量规律有独立解释力
7. **遥感特征**（5维）— 土地利用辅助，对细粒度功能区区分有补充价值
