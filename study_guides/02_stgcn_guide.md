# STGCN 论文导读

**论文**：Spatio-Temporal Graph Convolutional Networks: A Deep Learning Framework for Traffic Forecasting
**来源**：IJCAI 2018 | arXiv:1709.04875
**阅读时长**：约 60 分钟

---

## 读之前：一句话定位

> STGCN 是第一个同时用图卷积（处理空间）+ 纯卷积（处理时间）来做交通预测的模型，彻底抛弃了 RNN，训练速度快 14 倍。

---

## 一、背景痛点（论文 Introduction 翻译精华）

在 STGCN 出现之前，主流方法的问题：

| 方法 | 问题 |
|------|------|
| ARIMA / 统计方法 | 只能处理线性关系，无法建模复杂时空依赖 |
| CNN + 格网 | 要求数据是规则网格，无法处理任意图结构的传感器网络 |
| LSTM / GRU | 能处理时序，但不感知空间；逐步计算，不能并行，慢 |
| LSTM + GCN 交替 | 空间和时间分开处理，没有真正融合；RNN 仍然很慢 |

**核心洞察**：能不能用卷积（而不是 RNN）来处理时间维度？卷积可以并行，速度快得多。

## 二、模型架构逐层拆解

### 整体架构图（文字版）

```
输入 X: (时间步M, 节点N, 特征C)
         │
    ┌────▼────────────────────────┐
    │  ST-Conv Block 1            │
    │  时序卷积 → 图卷积 → 时序卷积 │  ← "三明治"结构
    └────────────────┬────────────┘
                     │
    ┌────────────────▼────────────┐
    │  ST-Conv Block 2            │
    │  时序卷积 → 图卷积 → 时序卷积 │
    └────────────────┬────────────┘
                     │
              时序卷积层
                     │
              全连接层 → 输出预测值
```

### 层 1：时序门控卷积（Gated Temporal Convolution）

**输入**：节点特征时间序列，形状 (M, C_in)

**做什么**：用 1D 卷积核（大小 K_t = 3）在时间轴滑动，提取时序特征

**关键公式**：

$$\Gamma *_{\mathcal{T}} Y = P \odot \sigma(Q)$$

**逐项解释**：
- `Γ`（Gamma）= 卷积核，形状 (K_t, C_in, 2*C_out)
- 卷积结果被均分为 P 和 Q 两部分（各 C_out 维）
- `P` = tanh 激活（特征提取）
- `σ(Q)` = sigmoid 激活（门控，控制哪些特征通过）
- `P ⊙ σ(Q)` = 门控输出（只有"被允许"的特征才能传下去）

**输出**：形状 (M - K_t + 1, C_out)，时间维度缩短了 K_t-1

### 层 2：图卷积（Chebyshev Graph Convolution）

**输入**：上一层输出，对图中每个节点独立处理

**做什么**：把相邻节点的特征聚合进来

**关键公式（Chebyshev 近似版本）**：

$$y_j = \sum_{i=1}^{C_i} \Theta_{i,j}(L) x_i \approx \sum_{k=0}^{K-1} \theta_k T_k(\tilde{L}) x$$

**逐项解释**：
- `T_k(L̃)` = 第 k 阶 Chebyshev 多项式（k=0 就是自身，k=1 就是一跳邻居，k=2 是两跳邻居）
- `θ_k` = 每阶的可学习权重
- K=3 意味着每个节点聚合自身 + 1跳邻居 + 2跳邻居的信息

**直觉**：把周围最多 K 步以内的邻居信息"拉"进来，就像 CNN 里的感受野。

### 层 3：第二个时序卷积

和层 1 一样，再做一次时序卷积。

**为什么要两头都加时序卷积？**

这就是论文里说的"三明治"（sandwich）结构：

```
时序卷积（捕捉时间特征）
    ↓
图卷积（融合空间信息，输入已经有时间特征了）
    ↓
时序卷积（再次整合，输出更丰富的时空特征）
```

空间和时间不是割裂处理的，而是相互增强的。

### ST-Conv Block 完整公式

$$v^{l+1} = \Gamma^l_1 *_{\mathcal{T}} \text{ReLU}(\Theta^l *_{\mathcal{G}}(\Gamma^l_0 *_{\mathcal{T}} v^l))$$

**逐项**：
- `v^l` = 第 l 层输入，形状 (M, N, C^l)
- `Γ_0 *_T v^l` = 第一个时序卷积
- `Θ *_G (...)` = 图卷积（在空间维度聚合邻居）
- `ReLU(...)` = 非线性激活
- `Γ_1 *_T (...)` = 第二个时序卷积
- `v^{l+1}` = 输出，形状 (M - 2*(K_t-1), N, C^{l+1})

## 三、为什么比 LSTM 快 14 倍？

| 特性 | LSTM | STGCN |
|------|------|-------|
| 时间步处理 | 逐步计算（必须等前一步完成） | 卷积并行处理所有时间步 |
| 参数量 | 大（GCGRU 参数量是 STGCN 的 1.5 倍） | 小（45万 vs 67万） |
| 梯度传播 | 容易消失（BPTT 问题） | 卷积结构更稳定 |

**数字对比（PeMSD7(M)）**：
- STGCN 训练时间：272 秒
- GCGRU（LSTM 变体）训练时间：3824 秒
- **快了 14 倍，精度还更高**

## 四、实验结果解读

**数据集**：PeMSD7（228个加州传感器）、北京道路数据

**评估指标**：
- MAE（平均绝对误差）= 预测值和真实值的平均偏差，单位是速度（km/h）
- MAPE = 相对误差百分比
- RMSE = 偏差的均方根，对大误差更敏感

**关键结论**：
- STGCN 在 15/30/45 分钟预测上均超过 LSTM 类方法
- 速度提升来自纯卷积架构
- 第一次验证了"图卷积 + 时序卷积"联合建模是有效的

## 五、STGCN 的局限性（读完后思考）

STGCN 留下了哪些问题供后续论文解决？

1. **图结构固定**：邻接矩阵在训练前定义好就不变了。但现实中，不同时间段节点间的关联是动态变化的（早高峰和深夜的交通模式完全不同）

2. **注意力缺失**：对所有邻居一视同仁，没有"这个邻居更重要"的机制

3. **长程依赖有限**：Chebyshev 近似只看 K 跳以内，K 通常很小（2-3），远处节点的信息传不进来

→ 这些问题正好是 **Graph WaveNet** 和 **Spacetimeformer** 要解决的。
