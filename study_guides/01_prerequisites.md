## 一、图（Graph）的基本概念

一个图 **G = (V, E, W)** 由三部分组成：
- **V**（节点 Vertices）：代表我们关心的实体。本项目中 = 城市网格或交通传感器
- **E**（边 Edges）：代表实体之间的关系。本项目中 = 相邻网格之间的空间连接
- **W**（权重矩阵 Adjacency Matrix）：N×N 的矩阵，`W[i][j]` 表示节点 i 和 j 之间的连接强度

**形象类比**：城市地图上有207个传感器，把每个传感器想象成一个节点，相邻传感器之间画一条线（边），线的粗细代表它们的物理距离近不近——这就是一个交通图。

### 图拉普拉斯矩阵（Graph Laplacian）

$$L = D - W$$

- **D** = 度矩阵（对角矩阵，`D[i,i]` = 节点 i 的边数之和）
- **L** = 拉普拉斯矩阵，编码了整个图的"连接结构"

**归一化版本**（论文里常用）：

$$\tilde{L} = D^{-1/2} L D^{-1/2} = I - D^{-1/2} W D^{-1/2}$$

**为什么要归一化？** 防止度数大的节点（很多邻居的节点）在计算中权重过大。

## 二、图卷积（Graph Convolution）

### 普通卷积 vs 图卷积

| | 普通 CNN 卷积 | 图卷积 GCN |
|---|---|---|
| 输入 | 规则网格（图片像素） | 不规则图结构（传感器网络） |
| 操作 | 滑动窗口 × 卷积核 | 邻居节点信息聚合 |
| 核心思想 | 局部特征提取 | 把邻居的特征"汇入"自己 |

### 图卷积的直觉

假设节点 A 想更新自己的特征，图卷积做的事情是：

```
新特征_A = 自己的特征 × 权重
           + 邻居B的特征 × 权重
           + 邻居C的特征 × 权重
           + ...
```

数学表示（矩阵形式）：

$$H^{(l+1)} = \sigma\left(\tilde{D}^{-1/2}\tilde{W}\tilde{D}^{-1/2} \cdot H^{(l)} \cdot \Theta\right)$$

- `H^(l)` = 第 l 层所有节点的特征矩阵（N × 特征维度）
- `Θ` = 可学习的权重矩阵（类似卷积核）
- `σ` = 激活函数（如 ReLU）
- `D^{-1/2}W D^{-1/2}` = 归一化邻接矩阵（做信息平均用的）

### Chebyshev 近似（STGCN 用到）

直接做上面的运算需要求解矩阵的特征值，计算量是 O(N²)，节点多时很慢。

**Chebyshev 近似** 用一系列多项式来代替，把复杂度降到 O(K × |边数|)：

$$\Theta *_{\mathcal{G}} x \approx \sum_{k=0}^{K-1} \theta_k T_k(\tilde{L})x$$

- `K` = 多项式阶数（相当于"看几跳邻居"，K=2 就是看2跳以内的邻居）
- `T_k` = 第 k 阶 Chebyshev 多项式（数学上的近似函数，不用细究）
- `θ_k` = 可学习参数

**一句话理解**：就是在每个节点上，把自己和2跳以内邻居的特征加权求和，权重可以训练。

## 三、注意力机制（Attention）

### 核心思想

给定一个问题（Query），在一堆信息（Keys + Values）里找最相关的。

**生活类比**：你去图书馆找"交通预测"相关的书（Query），图书馆员把所有书的标题（Keys）和你的问题对比打分，分高的书（Values）你就多看几眼。

### 数学公式

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

拆开理解：
- `Q`（Query）= 当前位置想"问"什么
- `K`（Key）= 其他位置"回答"什么
- `V`（Value）= 实际传递的信息内容
- `QK^T` = 点积，计算相关度得分（哪些位置最相关）
- `/ √d_k` = 缩放，防止点积值太大导致梯度消失
- `softmax` = 把得分变成概率（加起来等于1）
- `× V` = 按概率加权求和，取出最相关的信息

### 多头注意力（Multi-head Attention）

同时运行 H 个独立的注意力，然后把结果拼起来。

**为什么？** 一个注意力头只能关注一种关系，多个头可以同时捕捉"时间关系"、"空间关系"、"周期关系"等不同的模式。

### Self-Attention vs Cross-Attention

- **Self-Attention**：Q、K、V 都来自同一个序列（序列内部的位置互相关注）
- **Cross-Attention**：Q 来自一个序列，K、V 来自另一个序列（用于编码器-解码器之间的信息传递）

## 四、时序卷积（Temporal Convolution）

### 1D 卷积

对时间序列做卷积，一个卷积核覆盖连续几个时间步：

```
输入时间序列: [t-4, t-3, t-2, t-1, t]
卷积核大小 K=3: 每次覆盖3个时间步
输出: 提取局部时序特征
```

### 因果卷积（Causal Convolution）

预测时不能用"未来"的数据，所以卷积只向历史方向延伸：

```
t 时刻的输出只依赖 t, t-1, t-2, ... 的输入
```

### 空洞卷积/膨胀卷积（Dilated Convolution）

在卷积核位置之间插入间隔（dilation rate），用更少的参数覆盖更长的历史：

```
dilation=1: [t, t-1, t-2]       ← 覆盖3步
dilation=2: [t, t-2, t-4]       ← 覆盖5步，但只看3个点
dilation=4: [t, t-4, t-8]       ← 覆盖9步
```

**关键优势**：叠加多层后，感受野（能看到的历史长度）随层数指数增长，参数量却保持不变。

### 门控机制（Gating）GLU

$$h = \tanh(W_1 * x) \odot \sigma(W_2 * x)$$

- 左边 `tanh(...)` = 提取特征（值域 -1 到 1）
- 右边 `σ(...)` = 学习"这个特征有多重要"（值域 0 到 1）
- `⊙` = 逐元素相乘（门控：重要的通过，不重要的被压制）

## 五、残差连接（Residual Connection）

$$\text{输出} = \text{子层}(x) + x$$

**作用**：防止深层网络训练时梯度消失，让信息可以"直接跳过"某些层。

**形象理解**：爬楼梯时，有个电梯直通顶层。即使中间楼梯坏了（某层学到的特征没用），信息还是能通过电梯传到顶。
